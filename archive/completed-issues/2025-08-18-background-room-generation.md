# Background Room Generation System

**Date**: 2025-08-18  
**Status**: ✅ Completed  
**Priority**: High  
**Category**: Feature  

## Description

Implement background room generation to eliminate waiting time when players explore new areas. Instead of generating rooms on-demand when players move, pre-generate rooms in the background using AI to create a seamless exploration experience.

## Problem Statement

Currently, if we implement AI room generation naively, players would experience 2-3 second delays every time they try to enter an unexplored area while waiting for the Grok API to generate new content. This creates poor UX and breaks immersion.

## Solution: Adjacent Room Pre-Generation

**Strategy:** When a player enters any room, immediately start generating rooms for all adjacent areas in the background, creating a "bubble" of ready content around the player's current location.

**Core Logic:**
```
Player enters Room A
↓
Find all connections FROM Room A that lead to existing rooms
↓
For each connected existing room:
  - Check what directions from that room have no connections yet
  - Generate new rooms for those empty directions
↓
Continue gameplay (don't wait for generation to finish)
```

**Example Flow:**
```
Player enters Library
├── Library connects to Entrance Hall (exists)
│   └── Generate rooms for Entrance Hall's missing exits: down, west, secret
├── Library connects to Garden via bookshelf (exists)
│   └── Generate rooms for Garden's missing exits: north, up, fountain
└── Result: 5+ new rooms generated in background

Later when player goes to Garden → rooms are already waiting!
```


#### 2. Adjacent Room Expansion
```typescript
private async expandFromAdjacentRooms(currentRoomId: number): Promise<void> {
  try {
    // Get all connections FROM current room to existing rooms
    const connections = await this.db.all(
      'SELECT * FROM connections WHERE from_room_id = ? AND game_id = ?',
      [currentRoomId, this.currentGameId]
    );
    
    // For each connection that leads to an existing room
    for (const connection of connections) {
      const targetRoom = await this.db.get(
        'SELECT * FROM rooms WHERE id = ?', 
        [connection.to_room_id]
      );
      
      if (targetRoom) {
        // Generate rooms for this target room's empty exits
        this.generateMissingRoomsFor(targetRoom.id);
      }
    }
  } catch (error) {
    console.error('Background generation failed:', error);
    // Silent failure - game continues normally
  }
}
```

#### 3. Missing Room Detection and Generation
```typescript
private async generateMissingRoomsFor(roomId: number): Promise<void> {
  const allDirections = ['north', 'south', 'east', 'west', 'up', 'down'];
  
  for (const direction of allDirections) {
    // Check if connection already exists
    const existingConnection = await this.db.get(
      'SELECT * FROM connections WHERE from_room_id = ? AND name = ? AND game_id = ?',
      [roomId, direction, this.currentGameId]
    );
    
    if (!existingConnection) {
      // Generate new room in this direction (fire and forget)
      this.generateSingleRoom(roomId, direction);
    }
  }
}
```

#### 4. Individual Room Generation
```typescript
private async generateSingleRoom(fromRoomId: number, direction: string): Promise<void> {
  try {
    const fromRoom = await this.db.get('SELECT * FROM rooms WHERE id = ?', [fromRoomId]);
    
    const newRoom = await this.grokClient.generateRoom({
      currentRoom: { name: fromRoom.name, description: fromRoom.description },
      direction: direction
    });
    
    // Save to database
    const roomResult = await this.db.run(
      'INSERT INTO rooms (game_id, name, description) VALUES (?, ?, ?)',
      [this.currentGameId, newRoom.name, newRoom.description]
    );
    
    // Create connection
    await this.db.run(
      'INSERT INTO connections (game_id, from_room_id, to_room_id, name) VALUES (?, ?, ?, ?)',
      [this.currentGameId, fromRoomId, roomResult.lastID, direction]
    );
    
  } catch (error) {
    // Silent failure - this is background generation
    console.error(`Failed to generate room ${direction} from ${fromRoomId}:`, error);
  }
}
```

## Risk Assessment

### Risks
- **API costs** - Could generate many rooms if player backtracks frequently
- **Database bloat** - Unused rooms taking up space
- **Generation failures** - API outages affecting world expansion
- **Inconsistent content** - Generated rooms might not fit theme perfectly

### Mitigations
- **Cost control** - Rate limiting and generation budgets
- **Smart caching** - Track room usage, prioritize frequently visited areas
- **Fallback content** - Generic rooms if API unavailable  
- **Content validation** - Review and filtering of generated content
