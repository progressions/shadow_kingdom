# Issue Template

Use this template to create new issue files. Name files with format: `YYYY-MM-DD-issue-name.md`

## Issue Details

**Date**: YYYY-MM-DD  
**Status**: Open | In Progress | Resolved | Closed  
**Priority**: Low | Medium | High | Critical  
**Category**: Bug | Feature | Enhancement | Documentation | Performance  

## Description

Brief description of the issue or feature request.

## Details

Detailed explanation of:
- What is the problem/requirement?
- What should happen instead?
- Steps to reproduce (for bugs)
- Acceptance criteria (for features)

## Technical Notes

Any technical details, code snippets, or implementation notes.

## Resolution

*To be filled when issue is resolved*
- What was done to resolve the issue?
- Any additional notes or follow-up items?

## Related

- Links to related issues
- References to code files
- External resources
