# Combat UI System

**Date**: 2025-08-20  
**Status**: Open  
**Priority**: High  
**Category**: Feature  

## Description

Implement a comprehensive combat user interface that provides clear information display, intuitive action selection, and engaging combat feedback within the terminal-based game environment.

## Details

**What is the requirement?**
Create a combat UI system with the following components:

- **Combat Status Display**: Show all participants, HP, status effects, turn order
- **Action Menu System**: Clear options for player combat actions
- **Combat Log**: History of actions, damage, and effects
- **Turn Indicators**: Clear indication of whose turn it is
- **Damage/Healing Feedback**: Visual representation of HP changes
- **Status Effect Indicators**: Display temporary effects on participants
- **Victory/Defeat Screens**: Combat resolution summary

**Acceptance criteria:**
- [ ] Combat mode UI distinct from exploration mode
- [ ] Real-time HP display for all combat participants
- [ ] Clear action menu with keyboard shortcuts
- [ ] Combat log showing recent actions and results
- [ ] Turn order display with current actor highlighted
- [ ] Damage numbers and healing feedback
- [ ] Status effect indicators and duration tracking
- [ ] Victory screen with XP/loot summary
- [ ] Smooth transitions between combat and exploration UI

### Implementation Components

#### Combat Status Manager
```typescript
interface CombatUIState {
  participants: CombatParticipant[];
  currentTurn: number;
  round: number;
  combatLog: CombatLogEntry[];
  playerActions: CombatAction[];
  statusEffects: Map<string, StatusEffect[]>;
}
```

#### Display Functions
- `renderCombatHeader()`: Round info, participant overview
- `renderParticipantStatus()`: HP bars, status effects
- `renderCombatLog()`: Recent action history
- `renderActionMenu()`: Available player actions
- `renderDamageAnimation()`: Visual feedback for damage/healing

#### Terminal UI Integration
- Extend existing InkTUIApp components for combat mode
- Combat-specific message types for color coding
- Action input handling during combat turns
- Smooth transitions between UI modes

### UI Features

#### HP Bars and Indicators
- Visual HP bars using block characters: `[████████   ]`
- Color coding: Green (healthy), Yellow (injured), Red (critical)
- Percentage-based bar scaling

#### Status Effect Display
- Icons/symbols for different effects: ⚡ (energized), 🩸 (bleeding), 🛡️ (defended)
- Duration countdown display
- Effect description on hover/selection

#### Combat Log Management
- Scrollable history of combat actions
- Color coding for different action types
- Damage numbers highlighted
- Maximum log length with automatic cleanup

### Integration Points
- **TUIInterface**: Extend for combat mode switching
- **MessageType**: Add combat-specific message types
- **InputHandler**: Combat action input processing
- **StatusManager**: Combat state tracking
