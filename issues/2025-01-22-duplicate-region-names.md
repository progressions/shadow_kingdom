# Duplicate Region Names Issue

**Date**: 2025-01-22  
**Priority**: Medium  
**Status**: Open  
**Type**: Bug Fix  

## Problem Description

The region generation system is creating duplicate region names, leading to confusion and inconsistency in the game world. Multiple regions are being created with identical names but different IDs, making it difficult to distinguish between them.

## Current Behavior

Looking at the regions output:
```
- Night Bazaar (phantom market): 1 rooms
- Night Bazaar (phantom market): 1 rooms
- The Floating Archive (sky library): 4 rooms  
- The Floating Archive (sky library): 1 rooms
- Gloomspire Nexus (spire-network): 0 rooms
- Gloomspire Nexus (spire network): 0 rooms
```

The system is creating multiple regions with the same name, which breaks the immersion and logical consistency of the game world.

## Expected Behavior

Each region should have a unique name. When the system needs to create a new region, it should:

1. Check if a region with that name already exists in the current game
2. If it exists, either:
   - Use the existing region instead of creating a new one, OR  
   - Generate a unique variant name (e.g., "Night Bazaar East", "Night Bazaar Lower Level")
3. Ensure region names are truly unique within each game session

## Technical Context

- Region creation happens in the region generation service
- Region names are likely generated by AI without checking for duplicates
- This affects the `regions` command output and overall game consistency

## Acceptance Criteria

- [ ] No duplicate region names within a single game
- [ ] System checks for existing region names before creating new ones
- [ ] If duplicate name is generated, system creates a unique variant or reuses existing region
- [ ] All existing games continue to work (backward compatibility)
- [ ] Region uniqueness is maintained across game saves/loads

## Implementation Considerations

- May need to add uniqueness validation in region creation
- Consider if regions with same theme should be merged or given distinct names
- Ensure AI generation prompts can create name variants when needed
- Database constraints or application-level validation for region name uniqueness

## Files Likely Involved

- `src/services/regionService.ts` - Region creation and management
- `src/services/roomGenerationService.ts` - Integration with region system
- Region-related database operations
